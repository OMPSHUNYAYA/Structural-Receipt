Documentation Index and Repository Reading Guide

Release: v0.5.3
Status: Release guide


1 Versioning Policy

All public release artifacts in the current repository release use:

`v0.5.3`

Internal profile identifiers remain independently versioned identities.

Examples include:

`SR-CORE-5`

`SR-RESOLVER-5-D01`

`SR-CANON-1-D01`

`SR-EVIDENCE-BUNDLE-2-D01`

Those are profile identities, not competing project release versions.

The release rule is:

`public artifact version = v0.5.3`

`internal normative identity = applicable SR-* profile identifier`

3 Repository Functional Map

`demo/`

Interactive Structural Receipt creation, purpose resolution, lineage, evidence export, browser audit, and producer-side reconstruction.

`verify/`

Standalone browser verification, separate Python verification, frozen corpora, verification evidence, and the verification summary.

`bridge/`

Generic path-based JSON bridge and adapter manifest for provenance-preserving WRAPPED adoption.

`profiles/`

Nine standalone purpose-profile artifacts plus the purpose-profile set artifact.

`examples/`

Reproducible bridge and browser-to-verifier adoption examples.

`docs/`

Current public documentation, specifications, adoption guidance, and verification guidance.


4 Future Profile Boundary

Purpose-limited disclosure remains a future architecture direction and is not part of the frozen v0.5.3 implementation.

Current future profile directions include:

`SR-COMMIT-1-D01`

`SR-DISCLOSE-1-D01`

`SR-PDC-1-D01`

These identifiers describe proposed profile directions only.

They do not become part of the implemented release boundary until the applicable behavior is implemented, cross-implementation tested, documented, and frozen.


5 Public Claim Boundary

The documentation should preserve:

`RESOLVED != AUTHENTICATED`

`AUTHENTICATED != REAL-WORLD OCCURRENCE PROVEN`

`VERIFIED != APPROVED`

`APPROVED != EXECUTED`

`project-produced separate implementation != third-party certification`

Current third-party verification remains:

`NOT_CLAIMED`


6 Current Public Principles

`Declare once. Resolve by purpose. Preserve through change. Verify independently.`

`Adopt on one side. Verify on the other. Integrate only when useful.`
